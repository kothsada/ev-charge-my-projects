✦ ໄຟລ໌ panda-ev-nat-ip (34.126.166.249) ທີ່ທ່ານເຫັນນັ້ນ ແມ່ນ Cloud NAT IP Address. ມັນມີໜ້າທີ່ ແລະ ວິທີໃຊ້ທີ່ແຕກຕ່າງຈາກ Ingress IP ຫຼື LoadBalancer IP ດັ່ງນີ້:

  1. ມັນແມ່ນຫຍັງ? (What is it?)
   * Outbound Traffic Only: ມັນຖືກໃຊ້ສຳລັບໃຫ້ Pod ທີ່ຢູ່ໃນ Cluster (ເຊິ່ງຢູ່ພາຍໃນ Private Network) ສາມາດ "ອອກໄປຫາອິນເຕີເນັດ" ໄດ້.
   * ຕົວຢ່າງ: ເວລາ App ຂອງທ່ານຕ້ອງການສົ່ງ Notification ໄປຫາ Firebase, ຫຼື ຕ້ອງການເອີ້ນ API ຂອງ BCEL, ມັນຈະໃຊ້ IP 34.126.166.249 ນີ້ເປັນໂຕແທນໃນການສົ່ງຂໍ້ມູນອອກໄປ.

  2. ໃຊ້ແທນ Ingress ຫຼື OCPP LB ໄດ້ບໍ່?
   * ຄຳຕອບຄື: ບໍ່ໄດ້.
   * ເຫດຜົນ: NAT IP ຖືກອອກແບບມາໃຫ້ຂໍ້ມູນແລ່ນ "ອອກ" ເທົ່ານັ້ນ. ມັນບໍ່ຮອງຮັບການທີ່ຄົນພາຍນອກ (ຫຼື ຫົວສາກໄຟ) ຈະ "ເຂົ້າມາ" ຫາ Cluster ຜ່ານ IP ນີ້.

  3. ທ່ານຄວນເຮັດແນວໃດ?
  ສຳລັບ Ingress (Web API) ແລະ OCPP (WebSocket), ທ່ານຈຳເປັນຕ້ອງມີ IP ແຍກຕ່າງຫາກ:

  ສະຫຼຸບ:
   * 34.126.166.249 (NAT) = ໃຊ້ສຳລັບ App ສົ່ງຂໍ້ມູນອອກ ໄປຫາຂ້າງນອກ.
   * 34.8.243.174 (Ingress) = ໃຊ້ສຳລັບ User ເຂົ້າມາ ຫາ API.
   * OCPP IP (ຈອງໃໝ່) = ໃຊ້ສຳລັບ ຫົວສາກໄຟ ເຂົ້າມາ ສົ່ງຂໍ້ມູນ WebSocket.
