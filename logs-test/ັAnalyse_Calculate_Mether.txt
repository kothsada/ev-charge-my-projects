  energyKwh ທີ່ mobile ໄດ້ຮັບ ຖືກຄຳນວນໂດຍ backend:

  ທຸກຢ່າງຖືກ/ຜິດ ຂຶ້ນກັບ meterStart ທີ່ charger ສົ່ງໃນ StartTransaction message.

  ຈາກ log:
  Energy.Active.Import.Register ທຳອິດ = 17,840 Wh  (odometer ຂອງ charger)

  ສາກທີ 1 — Charger ສົ່ງ meterStart = 17,840 (ຖືກຕ້ອງ)

  energyKwh = (17,840 - 17,840) / 1000 = 0 kWh   ← ຖືກ

  ສາກທີ 2 — Charger ສົ່ງ meterStart = 0 (Bug ທີ່ Charger Firmware)

  energyKwh = (17,840 - 0) / 1000 = 17.84 kWh   ← ຜິດ! ທັນທີທີ່ session ເລີ່ມ

  ລູກຄ້າຈະເຫັນ 17.84 kWh ທັນທີທີ່ຊ໋ョ ットstart — ທຸກຄ່າ ຜິດໝົດ.

  ສາກທີ 3 — Charger ສົ່ງ meterStart ໃນ kWh (17.84) ແຕ່ MeterValues ໃນ Wh

  OCPP service ບໍ່ normalize meterStart (ກຳນົດໃຫ້ MeterValues ເທົ່ານັ້ນ):

  // ocpp.service.ts:315 — ເກັບຄ່າ as-is ບໍ່ມີ unit conversion
  meterStart: payload.meterStart,  // ⚠️  ຖ້າ charger ສົ່ງ 17.84 (kWh), ເກັບ 17.84

  // ແຕ່ MeterValues normalize → meterWh = 17,840 Wh
  energyKwh = (17,840 - 17.84) / 1000 = 17.822 kWh  // ← ຜິດ!

  ວິທີກວດສອບ: Query DB

  -- ກວດ meterStart ທີ່ charger ສ່ົງໃນ StartTransaction
  SELECT
    ocpp_transaction_id,
    meter_start,
    started_at,
    meter_stop,
    (meter_stop - meter_start) / 1000.0 AS energy_kwh_calculated
  FROM panda_ev_ocpp.transactions
  WHERE ocpp_transaction_id = 109;
